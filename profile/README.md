## Fixes for Common Connection Problems

- Connection refused? Confirm the hostname, protocol, and port. A winscp server profile for SFTP usually uses SSH port 22 unless the provider gives a custom value.
- Login failing? Recheck the username, password, private key, and account permissions. If winscp ssh authentication uses a key, make sure the correct private key file is selected.
- Transfers stalling? Test network stability, reduce simultaneous transfers, and check whether the remote server limits bandwidth or session count.
- Directory listing missing? Verify account permissions and remote path settings. Some hosting accounts restrict users to a specific starting folder.
- Platform confusion? Searches for winscp mac and winscp linux are common, but the official application is designed for Windows, so platform-specific alternatives may be needed outside Windows.
